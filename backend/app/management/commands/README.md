# Management Commands

This directory contains Django management commands for the OWSwims application. These commands handle event import, data maintenance, and administrative tasks.

## Quick Reference

| Command | Description |
|---------|-------------|
| `crawl_events` | Import events from websites using LLM-based processing |
| `discover_event_urls` | Discover new event URLs via Google Search |
| `update_next_year_events` | Update next year's events with fresh data from websites |
| `copy_to_next_year` | Copy events from one year to the next |
| `geocode` | Geocode locations without coordinates |
| `process_unverified_locations` | Process unverified locations (geocode + fetch images) |
| `merge_locations` | Merge duplicate locations within a distance |
| `merge_events` | Find and merge duplicate events at same location/date |
| `find_organizer_contacts` | Use AI to find organizer contact information |
| `inform_organizers` | Send emails to organizers |
| `list_crawl_profiles` | List available crawl profiles |
| `fix_race_currencies` | Fix race currencies based on country |
| `populate_organizer_slugs` | Generate slugs for organizers |
| `test_event_discovery` | Test the event discovery system |

## Event Import Commands

### `crawl_events`

The main command for importing events from websites using LLM-based processing (Firecrawl + OpenAI).

**Usage:**
```bash
# Process a single event from one or more URLs
python manage.py crawl_events --event https://example.com/event

# Process a single event from multiple URLs (same event)
python manage.py crawl_events --event https://example.com/event https://example.com/registration

# Crawl multiple events from a website
python manage.py crawl_events --crawl https://example.com/events

# Use a specific crawl profile
python manage.py crawl_events --profile outdoorswimmer

# Process from a discovered URLs JSON file
python manage.py crawl_events --discovered discovered_event_urls.json

# Process from a text file (one URL per line)
python manage.py crawl_events --file urls.txt
```

**Options:**
- `--event URL [URL ...]`: Process a single event from provided URLs
- `--crawl URL`: Crawl multiple events from a website
- `--profile NAME`: Use a specific crawl profile
- `--discovered FILE`: Process from a JSON file generated by `discover_event_urls`
- `--file FILE`: Process from a text file with one URL per line
- `--limit N`: Limit the number of events to process
- `--dry-run`: Process without saving to database
- `--custom-prompt TEXT`: Custom prompt for the agent (with `--profile` only)

**Environment Variables:**
- `FIRECRAWL_API_KEY`: Required for web scraping
- `OPENAI_API_KEY`: Required for LLM processing

### `discover_event_urls`

Discovers new open water swimming event URLs using Google Search and AI validation.

**Usage:**
```bash
# Basic usage (searches all countries)
python manage.py discover_event_urls

# Limit to specific countries
python manage.py discover_event_urls --countries CH DE FR

# Test run with limited queries
python manage.py discover_event_urls --limit 5 --dry-run
```

**Options:**
- `--limit N`: Limit the number of search queries
- `--output FILE`: Output file (default: `discovered_event_urls.json`)
- `--cache FILE`: Cache file for validation results (default: `url_validation_cache.json`)
- `--countries CODE [CODE ...]`: Limit to specific country codes
- `--dry-run`: Run without saving to file

**Environment Variables:**
- `OPENAI_API_KEY`: For GPT-4o validation
- `SERPER_API_KEY`: For Google Search
- `FIRECRAWL_API_KEY`: For web scraping

**Output Format:**
```json
[
  {
    "url": "https://example.com/event",
    "title": "Example Open Water Swim 2025",
    "snippet": "Join us for the annual open water swimming event...",
    "query": "open water swimming Switzerland 2025",
    "event_type": "single",
    "validation": "Valid open water swimming event"
  }
]
```

### `update_next_year_events`

Automatically updates next year's events by searching their websites for updated information.

**Usage:**
```bash
# Update events for 2026
python manage.py update_next_year_events 2026

# Dry run to preview changes
python manage.py update_next_year_events 2026 --dry-run

# Force check all events (ignore last check date)
python manage.py update_next_year_events 2026 --force
```

**Options:**
- `year`: Target year to update (required)
- `--limit N`: Limit number of events to process
- `--dry-run`: Preview without database changes
- `--check-interval N`: Skip events checked within N days (default: 30)
- `--force`: Check all events regardless of last check date

**How it works:**
1. Finds events for the target year that are invisible and unverified
2. Searches the event's website for updated information
3. Extracts and validates event data using LLM
4. Updates event details while preserving location and organizer

---

### `copy_to_next_year`

Copies events from one year to the next, creating placeholder entries for recurring events.

**Usage:**
```bash
python manage.py copy_to_next_year 2026
```

**Behavior:**
- Copies all visible events from the previous year
- Adjusts dates to the target year
- Marks new events as invisible and unverified
- Links to the previous year's event
- Copies associated reviews

## Location Commands

### `geocode`

Geocodes locations that don't have coordinates yet.

**Usage:**
```bash
python manage.py geocode
```

Uses Google Maps API to add latitude/longitude to locations missing coordinates.

---

### `process_unverified_locations`

Processes unverified locations by adding coordinates and header images.

**Usage:**
```bash
# Process all unverified locations
python manage.py process_unverified_locations

# Limit and auto-verify
python manage.py process_unverified_locations --limit 10 --auto-verify

# Dry run
python manage.py process_unverified_locations --dry-run
```

**Options:**
- `--limit N`: Limit number of locations to process
- `--auto-verify`: Automatically mark as verified after processing
- `--dry-run`: Preview without changes

---

### `merge_locations`

Finds and merges locations that are geographically close to each other.

**Usage:**
```bash
# Find locations within 500m of each other (default)
python manage.py merge_locations

# Custom distance threshold
python manage.py merge_locations --distance 1000

# Dry run
python manage.py merge_locations --dry-run
```

**Options:**
- `--distance N`: Maximum distance in meters (default: 500)
- `--dry-run`: Preview without changes
- `--limit N`: Maximum number of location groups to process

**Prioritization:**
1. Verified locations are preferred
2. Locations with images are preferred
3. Locations with more events are preferred

## Event Maintenance Commands

### `merge_events`

Finds and handles duplicate events at the same location on the same date.

**Usage:**
```bash
# Find all duplicates
python manage.py merge_events

# Check specific location
python manage.py merge_events --location 123

# Dry run
python manage.py merge_events --dry-run
```

**Options:**
- `--location ID`: Check only a specific location
- `--dry-run`: Preview without changes
- `--limit N`: Maximum number of duplicate groups to process

---

### `fix_race_currencies`

Fixes race currencies for future events based on country codes.

**Usage:**
```bash
# Fix all currencies
python manage.py fix_race_currencies

# Fix only US events
python manage.py fix_race_currencies --us-only

# Dry run
python manage.py fix_race_currencies --dry-run --verbose
```

**Options:**
- `--dry-run`: Preview without changes
- `--verbose`: Show detailed output
- `--us-only`: Only process US events

## Organizer Commands

### `find_organizer_contacts`

Uses AI to find contact information for organizers of future events.

**Usage:**
```bash
# Find contacts for organizers without contact info
python manage.py find_organizer_contacts

# Process all organizers (including those with contacts)
python manage.py find_organizer_contacts --all

# Process specific organizer
python manage.py find_organizer_contacts --organizer "Organizer Name"

# Reset and reprocess
python manage.py find_organizer_contacts --reset
```

**Options:**
- `--reset`: Reset existing contact info before processing
- `--organizer NAME`: Process only a specific organizer
- `--all`: Process all organizers including those with existing contacts

---

### `inform_organizers`

Sends informational emails to organizers with pending contact status.

**Usage:**
```bash
# Send emails (interactive mode)
python manage.py inform_organizers --interactive

# Dry run
python manage.py inform_organizers --dry-run

# Limit and customize
python manage.py inform_organizers --limit 5 --prompt-extension "Mention the new feature"
```

**Options:**
- `--dry-run`: Simulate without sending
- `--limit N`: Limit number of organizers
- `--prompt-extension TEXT`: Additional text for email generation
- `--interactive`: Confirm each email before sending

---

### `populate_organizer_slugs`

Generates URL slugs for organizers that don't have one.

**Usage:**
```bash
python manage.py populate_organizer_slugs
```

## Common Options

Most commands support these options:
- `--dry-run`: Preview changes without modifying the database
- `--limit N`: Limit the number of items to process
- `--verbose`: Show detailed output

## Environment Variables

Required for most commands:
```bash
FIRECRAWL_API_KEY=xxx      # Web scraping
OPENAI_API_KEY=xxx         # LLM processing
SERPER_API_KEY=xxx         # Google Search (discover_event_urls only)
GOOGLE_MAPS_API_KEY=xxx    # Geocoding and place images
```

Set these in `backend/.env`.
